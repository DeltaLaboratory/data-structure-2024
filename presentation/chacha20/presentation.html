현대 암호화 알고리즘의 특징과 성능

2024년 12월 2일

ChaCha20 소개

  • 2008년 Daniel J. Bernstein이 설계한 스트림 암호
  • Salsa20을 개선한 버전
  • 키 크기: 256비트
  • 논스(Nonce): 96비트
  • 카운터: 32비트
  • 주요 특징:
    • ARX(Addition, Rotation, XOR) 연산 기반
    • 소프트웨어 구현에 최적화
    • 병렬 처리 가능

AES 소개

  • 2001년 NIST 표준으로 채택
  • 블록 암호 방식
  • 키 크기: 128/192/256비트
  • 블록 크기: 128비트
  • 주요 특징:
    • SPN(Substitution-Permutation Network) 구조
    • 하드웨어 가속 지원
    • 광범위한 사용과 검증

블록 암호와 스트림 암호 비교

블록 암호 (AES)

주요 특징

  • 고정된 크기의 블록 단위로 처리
  • 일반적으로 128비트 블록
  • 패딩 필요
  • CBC, ECB, CTR 등 운영 모드 필요

장점

  • 높은 디퓨전(diffusion)
  • 병렬 처리 가능(일부 모드)
  • 하드웨어 가속 지원
  • 광범위한 검증

단점

  • 구현 복잡성
  • 패딩 오버헤드
  • 일부 운영 모드의 취약성

스트림 암호 (ChaCha20)

주요 특징

  • 비트/바이트 단위 처리
  • 키스트림 생성
  • 패딩 불필요
  • XOR 연산 기반

장점

  • 구현 단순성
  • 실시간 처리 적합
  • 오류 전파 없음
  • 메모리 효율성

단점

  • 키스트림 재사용 위험
  • 동기화 요구
  • 제한된 디퓨전

적용 시나리오

블록 암호 선호 스트림 암호 선호
  • 대용량 파일 암호화
  • 하드웨어 가속 가능 환경
  • 높은 보안성 요구 상황
  • 실시간 통신
  • 리소스 제한 환경
  • 스트리밍 데이터

주요 성능 특징

  • ChaCha20:
    • 하드웨어 가속 없이도 모든 플랫폼에서 안정적인 성능
    • AMD64에서 약 322 MB/s의 일관된 성능
    • 임베디드 환경(m0+)에서도 550.72 kB/s의 양호한 성능
  • AES:
    • 하드웨어 가속시 ChaCha20 대비 최대 7.5배 향상
    • 하드웨어 가속 없이는 ChaCha20보다 낮은 성능
    • 플랫폼별 성능 편차가 큼

결론: ChaCha20은 하드웨어 가속이 없는 환경에서 AES보다 우수한 성능을 보여주며, 특히 모바일이나 임베디드 시스템에서 효율적인 선택이 될 수 있다.

보안성 및 특징 비교

ChaCha20

  • 장점:
  • 사이드 채널 공격에 강함
  • 구현이 단순하고 안전
  • 하드웨어 의존성 낮음
  • 단점:
  • 상대적으로 새로운 알고리즘
  • 하드웨어 가속 지원 제한적

AES

  • 장점:
  • 광범위한 검증과 분석
  • 하드웨어 가속 지원
  • 산업 표준으로서의 신뢰성
  • 단점:
  • 최적화되지 않은 구현시 사이드 채널 취약
  • 소프트웨어 구현시 성능 저하 가능

결론

  • ChaCha20:
    • 모바일 및 소프트웨어 환경에 적합
    • 리소스 제한 환경에서 우수한 선택
    • 높은 보안성과 구현 용이성
  • AES:
    • 하드웨어 가속이 있는 환경에서 최적
    • 검증된 보안성과 광범위한 지원
    • 산업 표준으로서의 위치
